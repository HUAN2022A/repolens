# RepoLens v0.4.0

RepoLens v0.4.0 adds the first lightweight code intelligence layer: dependency-free symbol and import extraction.

## Highlights

- Extracts common symbols from JS/TS, Python, Go, Rust, and Java-family files.
- Extracts common import/dependency strings from supported languages.
- Adds repository-level symbol/import stats to `repo-map.json`.
- Adds per-file `symbols` and `imports` arrays to machine-readable output.
- Shows symbol and import hotspots in generated architecture and task context docs.

## Why it matters

RepoLens can now tell an agent not only which files matter, but also what important names and imports appear in those files. This is a stepping stone toward richer code graph and dependency analysis.

## Validation

- `npm test`
- `npm run smoke`
- `npm run pack:test`
- `npm pack --dry-run`
- GitHub Actions CI on Node.js 18 and 20

## Limitations

The analyzer is intentionally heuristic and dependency-free. It is not a full parser and may miss symbols or imports in complex syntax. AST/tree-sitter-based analysis remains a future direction.
