# RepoLens v0.5.0

RepoLens v0.5.0 adds the first internal dependency graph layer by resolving relative imports to repository files.

## Highlights

- Resolves relative imports such as `./utils` to likely repo files such as `src/utils.ts`.
- Supports common extension and `index.*` candidates.
- Adds `dependencyGraph.edges` to `repo-map.json`.
- Tracks unresolved relative imports separately.
- Adds incoming-edge hotspots to generated architecture docs.

## Why it matters

Agents can now see not just what a file imports, but whether that import connects to another file inside the repository. This is the first step toward a real code graph for task routing, review, and impact analysis.

## Validation

- `npm test`
- `npm run smoke`
- `npm run pack:test`
- `npm pack --dry-run`
- GitHub Actions CI on Node.js 18 and 20

## Limitations

- Dependency edges currently resolve relative imports only.
- Package imports, path aliases, tsconfig paths, and language-specific module systems are future work.
- The graph remains heuristic and dependency-free by design.
