# RepoLens v0.7.0

RepoLens v0.7.0 improves dependency graph resolution by supporting TypeScript/JavaScript path aliases.

## Highlights

- Resolves `tsconfig.json` and `jsconfig.json` `compilerOptions.paths` aliases.
- Supports the common `@/* -> src/*` convention when a repository has a `src/` directory.
- Marks dependency graph edges as `relative` or `alias`.
- Adds alias edge counts and alias rule metadata to `repo-map.json`.

## Example

```json
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "~/*": ["src/*"]
    }
  }
}
```

RepoLens can resolve:

```ts
import { api } from '~/lib/api'
```

to:

```json
{
  "from": "src/index.ts",
  "to": "src/lib/api.ts",
  "source": "~/lib/api",
  "kind": "alias",
  "alias": "~/"
}
```

## Validation

- `npm test`
- `npm run smoke`
- `npm run pack:test`
- `npm pack --dry-run`
- GitHub Actions CI on Node.js 18 and 20

## Notes

Package-level dependency resolution remains future work. RepoLens still keeps graph analysis dependency-free and heuristic.
