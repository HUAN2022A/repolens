# RepoLens v1.0.0

RepoLens v1.0.0 is the first stable GitHub release of the AI coding-agent repository context engine.

The v1.0 line turns RepoLens from a promising prototype into a usable local-first tool: a CLI, a minimal MCP server, task-aware file ranking, lightweight code intelligence, dependency graphing, impact analysis, test strategy guidance, release docs, and validation workflows.

## Highlights

- Stable `repolens` CLI for generating compact context packs.
- Stable `repolens-mcp` server exposing five agent tools:
  - `generate_context`
  - `repo_map`
  - `find_relevant_files`
  - `impact_analysis`
  - `test_strategy`
- Machine-readable `repo-map.json` for integrations.
- Markdown outputs for overview, architecture, task context, and paste-ready agent prompts.
- Heuristic symbol/import extraction across common languages.
- Internal dependency graph support for relative imports and common JS/TS path aliases.
- Impact and verification planning flows for AI coding agents.
- Architecture docs, MCP schema docs, roadmap, readiness audit, examples, benchmark notes, and community docs.

## Why it matters

AI coding agents are better when they start with the right context. RepoLens gives them a fast first pass: where to look, why those files matter, what might break, and how to verify the change.

This release is intentionally lightweight and local-first. It does not require a hosted service, a database, or heavyweight parser dependencies.

## Install

The package is prepared as `@huan2022a/repolens` while keeping the CLI commands short:

```bash
npm install -g @huan2022a/repolens
repolens . --task "fix payment webhook retries" --for codex
repolens-mcp
```

If npm has not been published yet, run from source:

```bash
git clone https://github.com/HUAN2022A/repolens.git
cd repolens
npm test
node src/cli.js . --task "understand this repo" --for codex
```

## Known limitations

- Symbol and import extraction is heuristic, not parser-backed.
- External package graphing and deeper monorepo workspace resolution are future work.
- MCP support is minimal stdio tooling; resources and prompts are planned for later.
- npm publication requires an authenticated npm session on the release machine.
